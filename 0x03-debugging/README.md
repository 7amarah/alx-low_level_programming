This project is an introduction to the concepts of Debugging.
It gives an overview of the "Rubber Duck Debugging" technique which takes advantage 
of the thought process that consciously teaching an inanimate objet or non-verbal being one's
code when it doesn't work as intended gives them insight to what truly went wrong.


The "Multiple mains" task requires the creation of a file named 0-main.c that tests that the function positive_or_negative() gives the correct output when given a case of 0.


The "Like, comment, subscribe" task requires the coping of a main file provided and then commenting out the part of the code that is causing the output to go into an infinite loop.


The "0 > 972?" task requires the fixing of the code in2-largest_number.c provided so that it correctly prints out the largest of three numbers, no matter the case.

The "Leap year" task requires the fixing of the print_remaining_days() function so that the output works correctly for all dates and all leap years.
